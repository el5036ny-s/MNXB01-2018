Soumettre une 'Pull Request' au dépôt 'patchwork' d'origine.

Pull Request

Souvent, lorsque vous effectuez des modifications et/ou des améliorations à un projet forké, vous souhaitez envoyer ces modifications à l'auteur d'origine. Pour ce faire, vous allez lui demander (request) qu'il tire (pull) vos modifications dans le dépôt original. C'est ce que l'on appelle une pull request dans le vocabulaire Git et ça permet partager ses mises à jour pour qu'elles bénéficient à tous.

Nous voulons vous ajouter à la liste des finisseurs d'atelier, alors faites une pull request pour ajouter le fichier avec votre nom d'utilisateur au dépôt original: github.com/jlord/patchwork.

Créer une pull request

Visitez le dépôt d'origine que vous avez forké sur GitHub, dans ce cas http://github.com/jlord/patchwork .

Souvent, GitHub détectera que avez poussé une branche sur votre fork et l'affichera en haut du site Web de l'original. Si vous voyez cela avec votre branche 'add-username', vous pouvez cliquer pour créer une 'pull request' à partir de là. Sinon:

  • Cliquez sur 'New pull request'.
  • Sélectionnez la branche avec les modifications que vous souhaitez soumettre. Ce devrait être 'add-yourusername'.

Vous verrez maintenant une page avec les détails de la pull request que vous êtes en train de soumettre. Cette page montre les 'commits' et les modifications, sous la forme d'un 'diff' par rapport au contenu de la branche 'gh-pages' de l'original.

Si le dépôt d'origine possède une documentation de contribution, GitHub affichera le lien vers celle-ci. C'est la documentation du propriétaires de ce dépôt sur la meilleure façon de contribuer à ce projet. C'est bien évidement très utile à lire et à respecter si vous souhaitez que vos changements soient adoptés!

Si tout sur la page semble correct :

  • Cliquez sur 'Create pull request'
  • Ajoutez un titre et une description aux modifications que vous proposez à l'auteur original.
  • Cliquez enfin sur 'Send pull request'!

Bingo! Vous avez soumis une 'pull request' prenez quelques secondes pour profiter du moment.

Si tout va bien avec votre pull request, elle sera fusionné en quelques instants. Si elle n'est pas fusionnée automatiquement dans les minutes qui suivent, vous aurez probablement des commentaires de @reporobot sur la raison pour laquelle il ne peut pas fusionner. Si c'est le cas, fermez votre pull request sur GitHub, effectuez les modifications nécessaires ett envoyez ces changements à votre fork et rouvrez (cela poussera @reporobot pour l'examiner à nouveau) votre pull request.
