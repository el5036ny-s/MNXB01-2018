Tirer depuis un Remote

Si vous travaillez sur quelque chose avec quelqu'un d'autre, vous devez toujours rester à jour avec les dernières modifications. Vous aurez donc besoin de tirer (pull) les modifications qui ont pu être poussées dans le dépôt central sur GitHub.

What has Reporobot been up to?

Voir si Reporobot a apporté des modifications à votre branche tirant les modifications de la remote 'origin' sur GitHub:

git pull <REMOTENAME> <BRANCHNAME>

Si rien n'a changé, il vous dira 'Already up-to-date'. S'il y a des changements, Git fusionnera ces changements dans votre version locale.

Mais qu'est ce que Reporobot a bien pu modifier? Git vous dit où des changements ont été apportés. Vous pouvez ouvrir le fichier et voir les mises à jour de Reporobot. Surprise, Reporobot est un artiste!

Sélectionner le dossier contenant votre dépôt.
  • Vérifier l'état de Git
    • git status
  • Tirer les modifications d'une branche distante
    • git pull <REMOTENAME> <REMOTEBRANCH>
  • Voir les modifications apportées à un remote avant de tirer
    • git fetch --dry-run
